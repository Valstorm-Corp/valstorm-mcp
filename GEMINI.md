# Valstorm MCP Server - AI Context

## Project Overview
This project is a Model Context Protocol (MCP) server for Valstorm. It provides a standardized interface for Large Language Models (LLMs, like Gemini or Claude) to interact directly with the Valstorm API. It acts as a secure proxy, executing queries, modifying records, and managing schemas on behalf of an authenticated user.

## Tech Stack
- **Framework:** `mcp` (Model Context Protocol Python SDK)
- **HTTP Client:** HTTPX
- **Package Manager:** uv
- **Validation:** Pydantic

## Core Architecture
- **Server Definition (`valstorm-mcp.py` / `main.py`):** Initializes the `FastMCP` server and registers the available tools.
- **Tools:** Python functions decorated with `@mcp.tool()`. These define the capabilities exposed to the LLM. Example tools include `run_sql_query`, `create_records`, `update_schema`, etc.
- **Authentication context:** The MCP server does not handle its own login flow. It relies entirely on the tokens generated by the `valstorm-cli`. It reads the target `VALSTORM_PROFILE` and `VALSTORM_ENV` environment variables to look up the correct `.json` auth file in `~/.valstorm/`.

## Development Guidelines
- **Adding New Tools:** 
  - Add a new async function decorated with `@mcp.tool()`.
  - **Crucial:** Provide an exhaustive, detailed Python docstring. The LLM relies entirely on the docstring and type hints to understand *when* and *how* to use the tool.
  - Keep arguments simple (strings, booleans, dicts, lists).
- **API Interactions:** 
  - Construct the HTTP requests using the base URL and auth headers derived from the environment.
  - Always handle HTTP errors gracefully, returning the error text as a string so the LLM can read the failure reason and adjust its strategy.
- **Idempotency & Safety:** Design tools so that if an LLM calls them multiple times (e.g., due to a timeout or retry loop), they behave safely or fail clearly.
